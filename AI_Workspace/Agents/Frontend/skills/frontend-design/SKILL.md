# frontend-design

## Objective
Provide practical guidance to improve visual hierarchy, layout rhythm, and product-facing polish in frontend implementations.

## Use when
- Building new pages or major UI sections.
- Refactoring visual design for clarity and consistency.
- Converting wireframes into production-ready UI.

## Core checks
1. Establish clear hierarchy (headline, support text, actions).
2. Keep spacing scale consistent across sections/components.
3. Ensure interactive states are explicit (hover, focus, disabled).
4. Validate responsive behavior on mobile + desktop.
5. Avoid decorative complexity that harms readability.

## Output expectations
- Explain key visual decisions briefly.
- Keep reusable style patterns in shared components/tokens when possible.
